## Sprint Structure

**Sprint Duration:** 2 weeks  
**Ceremonies:**
- Sprint Planning (Day 1)
- Daily Standups
- Sprint Review (Last Day)
- Sprint Retrospective (Last Day)

**Definition of Done:**
1. Code reviewed and merged
2. Tests passing (unit + feature)
3. Responsive on mobile/tablet/desktop
4. Dark mode verified
5. No linter errors
6. Documentation updated
7. Deployed to staging
